# UI変更後の確認手順

UIコンポーネントの実装・変更を行った後は、以下の手順に従って確認を行うこと。

## 原則

- コードから表示結果を推測しない。必ずブラウザでレンダリングされた実際の画面をスクリーンショットで確認する
- 「このコードならこう表示されるはず」という判断は禁止。実際の描画結果のみを信頼する

## 確認手順

1. UI変更後、ブラウザでページを開きスクリーンショットを撮影する
2. 変更箇所が意図通りに描画されているかスクリーンショットで確認する
3. 差異や問題があれば修正し、再度スクリーンショットで確認する
4. スクリーンショットで確認が完了するまで「完了」と報告しない

## スタイル値の扱い

- 色、サイズ、フォント、余白等のスタイル値を設定する際は、指定された値や参照元の実値を使用する。推測で値を決めない
- 参照元がある場合（既存サイト、デザインカンプ等）は、DevToolsのcomputed styleやソースコードから実値を取得する
- computed styleを取得する際は、コンテナ要素ではなく実際のテキスト要素やターゲット要素を正確に特定する

## HTML構造の確認

- 複数の要素が1つのカード/パネルにまとまっているか、別々に分かれているかを確認する
- 背景色・ボーダー・border-radiusが適用される範囲（どの親要素に設定されているか）に注意する
